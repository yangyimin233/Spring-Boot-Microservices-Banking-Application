## 1. 登录鉴权

### 正常登录

```bash
curl -s -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"emailId":"zhangsan@test.com","password":"123456"}'

# 返回: { "token": "eyJhbGciOiJIUzI1NiJ9..." }
```

**预期结果:** HTTP 200，返回 JWT token

### 密码错误

```bash
curl -s -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"emailId":"zhangsan@test.com","password":"wrong"}'

# 预期: HTTP 401 Unauthorized
```

### 无 Token 访问受保护接口

```bash

  -H "Content-Type: application/json" \
  -d '{"accountId":"0600140000001","amount":100,"transactionType":"DEPOSIT"}'

# 预期: HTTP 403 Forbidden
```

## 3. 存款接口

### 3.1 正常存款

```bash
TOKEN="<zhangsan的token>"

curl -s -w "\nHTTP:%{http_code}" -X POST http://localhost:8080/transactions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "accountId":"0600140000001",
    "amount":1000,
    "transactionType":"DEPOSIT",
    "description":"测试存款"
  }'

# 预期: HTTP 201, { "responseCode":"200", "message":"Transaction completed successfully" }
```

### 3.2 幂等验证 — 同 referenceId 重复发送

```bash
curl -s -X POST http://localhost:8080/transactions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "accountId":"0600140000001",
    "amount":100,
    "transactionType":"DEPOSIT",
    "referenceId":"IDEMPOTENT-TEST-001"
  }'

# 第二次发送同一个请求 → 返回 200，message 含 "(idempotent)"
# 分录表里只有 2 条（1 张凭证），余额只增 1 次
```

**验证:** 查分录表 `SELECT * FROM journal_entry WHERE account_id='0600140000001' ORDER BY id DESC LIMIT 5;` — 同一 referenceId 只有一次记录。

### 3.3 水平鉴权 — 张三用李四的 accountId

```bash
# 张三的 token，但 accountId 用李四的
curl -s -w "\nHTTP:%{http_code}" -X POST http://localhost:8080/transactions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "accountId":"0600140000002",
    "amount":100,
    "transactionType":"DEPOSIT"
  }'

# 预期: HTTP 403 Access Denied
```

## 4. 取款接口

### 4.1 正常取款

```bash
curl -s -w "\nHTTP:%{http_code}" -X POST http://localhost:8080/transactions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "accountId":"0600140000001",
    "amount":300,
    "transactionType":"WITHDRAWAL"
  }'

# 预期: HTTP 201，取款成功
```

### 4.2 余额不足

```bash
# 余额假设 700，取 800
curl -s -w "\nHTTP:%{http_code}" -X POST http://localhost:8080/transactions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"accountId":"0600140000001","amount":800,"transactionType":"WITHDRAWAL"}'

# 预期: HTTP 400, "Insufficient balance in the account"
```

### 4.3 幂等验证

同存款，带 `referenceId` 重复发送，第二次返回幂等响应。

### 4.4 水平鉴权

张三 token + 李四 accountId → 403 Access Denied（同存款）。

### 4.5 并发超扣校验（JMeter）

```
场景: 余额 700，10 个线程同时取款 500

JMeter 配置:
  - Thread Group: 10 threads, Ramp-Up 0s, Loop 1
  - HTTP Request: POST /transactions
  - Body: {"accountId":"0600140000001","amount":500,"transactionType":"WITHDRAWAL"}
  - Authorization Header: Bearer <token>

预期结果:
  - 10 次请求: 最多 1 次 HTTP 201（成功）
  - 其余返回 400（Insufficient balance 或 系统繁忙）
  - 账户余额绝不为负数
  - journal_entry 表最多 2 条新分录（1 张取款凭证）
```

## 5. 转账接口

### 5.1 正常转账

```bash
curl -s -w "\nHTTP:%{http_code}" -X POST http://localhost:8080/fund-transfers \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "fromAccount":"0600140000001",
    "toAccount":"0600140000002",
    "amount":100
  }'

# 预期: HTTP 200, { "transactionId":"TX2026062600000001", "message":"Fund transfer successful" }
```

### 5.2 余额不足

```bash
# 张三余额假设 100，转 500
curl -s -w "\nHTTP:%{http_code}" -X POST http://localhost:8080/fund-transfers \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"fromAccount":"0600140000001","toAccount":"0600140000002","amount":500}'

# 预期: HTTP 400, "Insufficient balance for account: 0600140000001"
```

### 5.3 幂等验证

转账的 `transactionReference` 由服务端自动生成（从 Sequence-Generator 获取），单次请求天然幂等。

如需验证客户端幂等，可多次调用 `/transactions/internal` 携带相同 `transactionReference`。

### 5.4 水平鉴权

张三 token + 李四的 fromAccount → 403 Access Denied。

### 5.5 并发超扣校验（JMeter）

```
场景: 张三余额 100，李四余额 1000，10 个线程同时张三→李四转 50

JMeter 配置:
  - Thread Group: 10 threads, Ramp-Up 0s
  - HTTP Request: POST /fund-transfers
  - Body: {"fromAccount":"0600140000001","toAccount":"0600140000002","amount":50}

预期结果:
  - 最多 2 次成功（余额 100 / 50 = 2 次）
  - 其余返回 Insufficient balance
  - 张三余额不为负
  - 李四余额增量 = 成功次数 × 50
  - 转账日志 fund_transfer 表记录数 = 成功次数
```

### 5.6 死锁测试（JMeter）

```
场景: 张三→李四 10 元 和 李四→张三 5 元，两组各 10 线程并发

JMeter 配置:
  - Thread Group 1: 张三→李四 10 元, 10 threads
  - Thread Group 2: 李四→张三 5 元, 10 threads
  - 同时启动

预期结果:
  - 所有请求正常处理，无死锁
  - 无请求超时或永久挂起
  - 账户余额逻辑正确
```

## 6. 实时余额核算接口

```bash

curl -s http://localhost:8084/transactions/internal/balance?accountId=0600140000001

# 返回: 1500.0000
```

### 6.2 Benchmark 对比

```bash
curl -s http://localhost:8084/transactions/internal/balance/benchmark?accountId=0600140000001

# 返回:
# {
#   "runningBalance_value": 1500.0000,
#   "runningBalance_ms": "0.450",    ← O(1) 走 running_balance
#   "sum_value": 1500.0000,
#   "sum_ms": "5.200",                ← O(n) 走 SUM
#   "speedup": "11.6x",
#   "match": true                     ← 两条路径一致
# }
```
